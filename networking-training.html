Section 01

Ethernet Cables

  • Connect devices to a network using copper wire
  • Use RJ45 connectors — the plastic clip on each end
  • Can carry both data and power (PoE)
  • Ethernet uses 4 twisted pairs — 8 wires total
  • A link light ≠ a good cable. One bad wire can drop speeds or cause intermittent faults
⚠ Most Common Fault: Bad cable terminations. Even pre-made patch leads can be faulty. One bad wire out of 8 can drop speed from 1 Gbps to 100 Mbps — and it will still show a link light.
CAT Rating — Speed & Distance
CAT5e1 Gbps @ 100m
CAT61 Gbps @ 100m  |  10 Gbps @ 55m
FibreLong distance — 100m+
Fibre Note: Use fibre when distances exceed 100m. Fibre has a minimum bend radius — don't kink it. Formulas for this are available online.
Section 02

Troubleshooting

Golden Rule: Always start at the physical layer. Check connections, rule out hardware, reduce the number of possible problem points.

  1. Simplify the setup. Disconnect from building cabling. Use a PoE injector directly into the router — bypass as much hardware as possible.

  2. Check every cable. Swap patch leads. Test with a cable tester if available. Remember — a link light doesn't mean a good cable.

  3. Check power. Confirm PoE is actually delivering power. Try a different port or a direct PoE injector.

  4. Check IP / subnet. Is your laptop on the same network as the device? (More on this next.)

Example Scenario

Access Point Not Working

Your AP is: plugged into a data outlet → connected to a PoE switch → linked to a router.

➡ Simplify it:

  • Disconnect from building cabling
  • Plug directly into router with a PoE injector
  • Does it work now? → The problem is in the building cabling or switch
  • Still broken? → Problem is the AP itself or the router
Section 03

IP Addresses & Subnets

An IP address is simply a device's unique address on a network — just like a house needs a street address so mail knows where to go.

Networking

🌐Network = The City
🔀Subnet = The Street
💻Device = The House
🔢IP Address = House Number
🚦Router = Intersection

Electrical

🏗️Network = Switchboard
⚡Subnet = Circuit
🔌Device = Outlet / Load
🔗Router = Controlled link between circuits
⚠ Common Real-World Issue

Can't Find the Camera?

Installer connects laptop to WiFi. Camera is on a different VLAN or subnet.

Result: Camera can't be discovered — not because anything is broken, but because they're on different subnets. You need the router to bridge them, or you need to be on the same subnet.

Key rule: Devices on the same subnet see each other directly. Devices on different subnets need a router to communicate.
Section 04 — Network Devices

The Router

  • The front door of your network — controls what comes in from the outside world
  • Handles security and firewall rules
  • Connects directly to the fibre box (ONT) on the wall
  • Assigns IP addresses to all devices on the network (via DHCP)
  • Plug the ONT into the WAN port, everything else into any other port
Think of it as: The main switchboard of the network. All traffic in and out passes through the router.
NZ Fibre Special Note 🇳🇿

Configure Before Use

NZ fibre connections use VLAN tags on the WAN connection — routers don't work out of the box. You must configure this first.

Spark adds PPPoE on top — requires a username and password:

user@spark.co.nz
password: password

✅ Starlink is the exception — works normally out of the box.

Section 04 — Network Devices

The Switch

  • Splits a single cable into multiple ports — gives you more connections
  • Plug internet into any port (avoid ports labelled Console or MGMT)
  • Many switches include PoE capability to power devices
Think of it as: A power board for your network — one cable in, many devices connected.
Type 1

Unmanaged Switch

Plug and play. No configuration required. Good for simple home or small installations where you just need more ports.

Type 2

Managed Switch

Used in business/commercial installs. Allows you to separate traffic — e.g., keep guests off the camera network using VLANs. Requires configuration.

Section 04 — Network Devices

Access Points & PoE

Access Point (AP)

Wireless Coverage

  • Provides WiFi to wireless devices
  • Continues to broadcast WiFi even without internet — you can still connect and configure it without a laptop cable
  • All Grandstream APs are PoE powered — via switch or injector
  • Wired to router is always preferred. Wireless mesh is available but can cause issues
PoE Injector

Power Over Ethernet

  • One LAN input, one PoE output
  • Delivers power and data over a single cable
  • Mostly AC powered — DC options exist
  • Great for troubleshooting — bypass the switch and inject PoE directly
Wall power → PoE Injector → One cable → Access Point
↑ Power + Data in a single run
Section 05

Wi-Fi Standards & Frequencies

WiFi Standards — Higher = Better
WiFi 5 (ac)Common, solid performance
WiFi 6 (ax)Faster, more reliable, better security
WiFi 7 (be)Latest — highest speeds
dBi (Signal Strength) is logarithmic — 9 dBi is 2× as powerful as 6 dBi. Directional antennas are always stronger than omnidirectional at the same dBi rating.
Frequency Bands — Trade-offs
2.4 GHz
Best range
5 GHz
Sweet spot
6 GHz
Short range

Higher frequency = faster speed + better reliability, but less penetration through walls and obstacles. Like light — red travels further than blue.

Section 06

🇳🇿 NZ Fibre Setup

New Zealand set up its fibre network differently to most of the world. This means routers do not work out of the box — you must configure them first.

  • NZ fibre uses VLAN tags on the WAN connection — must be set in the router
  • Spark uses PPPoE — requires a username & password in the router settings
  • Starlink is the exception — works normally without extra configuration
Spark PPPoE Credentials

Common Trap

If the client is on Spark and the router isn't configured for PPPoE, you'll get no internet even with everything physically correct.

Username: user@spark.co.nz
Password: password

Always check ISP and configure WAN settings before assuming there's a hardware fault.

Section 07

Point to Point Links

An invisible wire through the air — can carry internet up to 1–50km at high speeds. Think: sleep out 100m away that needs Netflix.

  • 500 Mbps over 1km available from ~$200
  • Requires direct line of sight — think a rugby ball-shaped clear zone, not just a gap in trees
  • Faster speeds = thinner "rugby ball" = shorter effective distance
  • Lower frequencies (2.4 GHz) handle obstacles better than higher ones (60 GHz)
⚠ Frequency vs Obstacles

Choose Wisely

2.4 GHz — handles obstacles, best for marginal LoS
5 GHz — sweet spot for most P2P installs
60 GHz — killed by rain, oxygen, leaves. True clear LoS only
Unofficial tip: You can sometimes get away with partial LoS — but it will hurt performance and distance significantly. When in doubt, choose a lower frequency and recommend a site survey first.
Wrap Up

Key Takeaways

01

Physical First

Always start troubleshooting at the physical layer. Cables cause most network problems. A link light ≠ a good cable.

02

Same Subnet = Visible

Devices on different subnets can't see each other without a router. Check this before assuming something is broken.

03

Simplify to Diagnose

Remove as much hardware from the equation as possible. Plug directly into the router with a PoE injector to isolate the fault.

04

NZ Needs Config

NZ routers don't work out of the box — VLAN tags required. Spark needs PPPoE credentials too.

05

Frequency Trade-offs

Higher frequency = faster but shorter range. 5 GHz is the sweet spot for most installs. 2.4 GHz for range or obstacles.

06

LoS for P2P

Point-to-point links need true line of sight — not just a gap. Choose frequency based on obstacles and distance.
